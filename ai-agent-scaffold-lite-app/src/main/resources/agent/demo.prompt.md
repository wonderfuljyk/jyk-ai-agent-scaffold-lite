根据d emo.yml 描述模板，按照我的要求配置一个 agent-draw-io.yml 智能体。用户和智能体对话，发起绘图诉求。要求如下；

1. 智能体返回的消息要有json结构，类型区分为 user、drawio。user 则是让用户继续回复补充信息，drawio 则是把绘图的内容，直接让 draw.io 面板渲染出来。
2. 如果智能体不能确认，用户具体的绘图诉求，则需要反馈信息给前端，让用户补充信息。
3. 用户的请求信息，要进行检索，分析，决策，执行，得到一个最准确的绘图内容，包括绘制uml图、流程图等各类图。如果用户配置的mcp，具备一些git仓库，本地检索文件等能力，也要根据用户的请求进行使用。
4. 之后要进行绘图操作，绘图后要在检查，绘图内容连线有没有交叉等问题。

综上，基于这样的场景，要区分出来单一智能体，之后可以通过循环、并行、串行配置，用于达到最佳的绘图效果。
